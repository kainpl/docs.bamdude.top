# Файловий менеджер

Переглядайте, завантажуйте та керуйте файлами у локальній бібліотеці BamDude. Друкуйте напряму або додавайте до черги.

---

## :material-folder: Огляд

Файловий менеджер дозволяє:

- **Переглядати** файли у локальній бібліотеці
- **Монтувати зовнішні папки** з NAS, USB або мережевих ресурсів
- **Завантажувати** файли, включаючи ZIP-архіви
- **Друкувати напряму** на будь-якому принтері
- **Додавати до черги** нарізані файли для подальшого друку
- **Перейменовувати** та **видаляти** файли і папки

---

## :material-printer: Прямий друк

1. Знайдіть нарізаний файл (`.gcode` або `.gcode.3mf`)
2. Натисніть іконку принтера або клацніть правою кнопкою для контекстного меню
3. Виберіть **Print**
4. Оберіть принтер(и), налаштуйте відповідність філаментів, задайте параметри друку
5. Натисніть **Print** для запуску

### Додавання до черги

Додавайте нарізані файли до черги для подальшого друку без попереднього створення архівів. Архіви створюються автоматично при запуску друку.

---

## :material-folder-zip: Завантаження ZIP-файлів

Завантажуйте ZIP-архіви для розпакування вмісту у бібліотеку:

1. Натисніть **Upload** та виберіть `.zip` файл
2. Оберіть, чи зберігати структуру папок
3. Натисніть **Extract**

---

## :material-database: Як зберігаються файли

Кожен файл бібліотеки -- це рядок у таблиці `library_files`. Рядок несе:

- **Hash-дедуп** -- завантажені файли SHA-256-яться і зіставляються з наявними рядками; ідентичний повторний upload повертає існуючий запис, замість створення дубля копії на диску.
- **Мініатюри** -- витягуються з `Metadata/plate_*.png` всередині 3MF при завантаженні (без on-the-fly екстракції). Повторний upload або "reparse" тригерять нову екстракцію.
- **`print_count` + `last_printed_at`** -- лічильники використання, які підтримує dispatch; видно при наведенні на картку файла, використовуються режимами сортування. Заповнюються ретроактивно при апгрейді через міграцію `m014`.
- **JSON-колонка `file_metadata`** -- зберігає розпарсені metadata слайсера: ваги філаменту по котушках, кількість об'єктів, цільова модель принтера в слайсері, плюс нові прапорці `gcode_label_objects` / `exclude_object` з `Metadata/project_settings.config` вихідного 3MF (витягуються в 0.4.1, заповнюються міграцією `m022`). Прапорці label-objects є воротами для кнопки **skip-objects** на сторінці принтера під час друку -- обидва мають бути `true`, щоб кнопка засвітилася. Bambu Studio вмикає обидва за замовчуванням; OrcaSlicer постачає обидва вимкненими (див. [Troubleshooting](../reference/troubleshooting.md) для чек-листа на стороні слайсера).
- **Прапорець `swap_compatible`** -- визначається з суфікса імені файла (`_swap`, `_swap.gcode.3mf` і подібні). Swap-сумісні файли окремо виносяться в swap-mode picker.

---

## :material-link-variant: Лінки на проєкти та папки

- **Лінк на папку** -- лінкування папки до проєкту проставляє `project_id` кожному файлу всередині, і будь-який файл, переміщений у цю папку пізніше, успадковує лінк.
- **Лінк на файл** -- кожен рядок файла також має власну кнопку `Link2`, щоб приєднати його до проєкту незалежно від папки.
- **Plan items проєкту** (m016) -- сторінка проєкту рендерить плаский plan-список з копіями/порядком/підсумками; рядки з'являються автоматично, коли файли / папки лінкуються до проєкту, і per-row підсумки (філамент, час, вартість) живлять загальні підсумки проєкту.

---

## :material-folder-network: Монтування зовнішніх папок

Монтуйте директорії хоста (NAS, USB-накопичувачі) у файловий менеджер без копіювання файлів:

1. Підключіть директорію через bind-mount у Docker
2. Натисніть **Link External** на панелі інструментів
3. Введіть відображувану назву та шлях у контейнері
4. Файли індексуються та з'являються негайно

---

## :material-lightbulb: Поради

!!! tip "Підтримка кількох принтерів"
    Виберіть кілька принтерів, щоб надіслати один файл на всю ферму одночасно.

!!! tip "Позначки файлів"
    Шукайте позначки "sliced", щоб визначити файли, готові до друку.

> Базується на документації [Bambuddy](https://github.com/maziggy/bambuddy).
