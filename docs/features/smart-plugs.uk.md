# Розумні розетки

Керуйте принтерами за допомогою розумних розеток Tasmota, Home Assistant, REST/Webhook або MQTT для моніторингу живлення, автоматизації та відстеження енергоспоживання per-print.

---

## :material-power-plug: Огляд

Інтеграція з розумними розетками забезпечує:

- **Управління живленням** -- дистанційне увімкнення/вимкнення принтерів
- **Моніторинг енергії** -- відстеження споживання (lifetime + per-print)
- **Авто-увімкнення** -- запуск принтера перед запланованим друком
- **Авто-вимкнення** -- вимкнення після охолодження

| Тип | Управління | Енергія | Опис |
|-----|:----------:|:-------:|------|

## :material-robot: Автоматизація

### Авто-увімкнення

Коли друк з черги готовий до запуску, BamDude вмикає розетку, чекає на завантаження принтера, а потім стартує друк.

### Авто-вимкнення

Після завершення друку BamDude чекає охолодження стола, перевіряє наявність інших друків у черзі, а потім вимикає живлення.

Налаштовується в **Settings → Smart Plugs** з параметрами температури охолодження та часу.

## :material-flash: Відстеження енергії per-print

Для розумних розеток, що репортують kWh-лічильник, BamDude знімає показання лічильника на старті друку та ще раз на завершенні. Дельта -- це енергія, спожита **саме цим друком**, збережена в рядку архіву.

**Як це працює**

- Друк стартує → BamDude читає поточний kWh розетки та пише його в `PrintArchive.energy_start_kwh`.
- Друк завершується → свіжа сесія БД перечитує розетку, обчислює `current - energy_start_kwh` і зберігає результат на архіві.
- Невдалі та скасовані друки фіксують часткову енергію -- дельта від старту до перерви все одно осмислена.

**Restart-стійкість**

`energy_start_kwh` зберігається в рядку архіву, ніколи не тримається в in-memory dict. Рестарт BamDude посеред друку зберігає стартовий baseline, і print-end-хендлер все одно видасть коректну дельту.

**Lifetime-агрегація**

Для звітів типу "kWh, спожиті принтером X цього місяця", BamDude робить погодинні снепшоти у таблицю `smart_plug_energy_snapshots`. Тотали по діапазону дат рахуються per-plug як `max(0, last_in_range - baseline)`, тож скидання лічильника (перепрошивки firmware, перезавантаження розетки по живленню) не дає від'ємних дельт.

### Energy Display Mode

У **Settings → System → "Energy display mode"**:

| Режим | Джерело | Сценарій |
|-------|---------|----------|
| `print` | Сума архівних дельт per-print | "Скільки електрики коштували друки, які я зробив?" |
| `total` (за замовчуванням) | Lifetime-лічильник розетки через snapshot-діапазон | "Скільки спожила розетка принтера?" |

Два режими можуть розходитися, коли принтер під живленням (нагрів, idle, standby) без активного друку -- лише `total` це ловить.

## :material-wrench-cog: Надійність та обслуговування

Розумні розетки авто-перепідписуються в MQTT при кожному старті BamDude через хелпер `subscribe_plug_to_mqtt`. Більшість випадків "розетка перестала відповідати" фіксяться одним рестартом.

**Якщо розетка не відповідає після рестарту**

1. Підтвердьте, що розетка працює у власному застосунку (web UI Tasmota / HA dashboard / Tasmota Console).
2. Перевірте конфіг MQTT-топіків розетки в **Settings → Smart Plugs** -- per-type поля мають збігатися з реальними топіками брокера.
3. Рестартніть BamDude ще раз після виправлення топіка.

Startup-restore code path, create-роут і update-роут усі ходять через той самий хелпер, тож конфігурація топіків не може розходитися між create та reconnect.

## :material-lightbulb: Поради

!!! tip "Почніть просто"
    Спочатку використовуйте ручне керування живленням, перш ніж вмикати
    автоматизацію. Назбирайте впевненість у надійності розетки, перш ніж
    давати BamDude циклити живлення сам.

!!! tip "Перевірте охолодження"
    Прогляньте кілька друків, щоб знайти оптимальну температуру охолодження
    для вашого принтера.

!!! tip "Паруйте з макросами"
    Поєднайте авто-вимкнення живлення з макросом `print_finished`, щоб
    одночасно вимикати світло корпусу. Див. [Макроси](macros.md).

!!! tip "Звітність по енергії"
    Перемкніть energy display mode на `print`, якщо ви рахуєте з клієнтів
    per-print. Використовуйте `total` для особистого "скільки коштує моя
    ферма" -- він включає idle/standby споживання, яке `print` пропускає.

> Початково базується на документації [Bambuddy](https://github.com/maziggy/bambuddy).
