# Telegram-бот

BamDude включає вбудованого Telegram-бота (aiogram 3.x) для повного керування принтерами, моніторингу та сповіщень безпосередньо з Telegram.

---

## :material-robot: Огляд

Telegram-бот забезпечує:

- **Статус принтерів** -- Перегляд усіх принтерів зі статусом у реальному часі
- **Керування друком** -- Пауза, зупинка та відновлення друків
- **Знімки з камери** -- Перегляд живих зображень з камери
- **Керування швидкістю** -- Зміна швидкості друку на льоту
- **Друк з бібліотеки** -- Перегляд та запуск друків з файлової бібліотеки
- **Керування чергою** -- Перегляд та управління чергою друку
- **Калібрування** -- Запуск процедур калібрування принтера
- **Обслуговування** -- Перегляд та виконання завдань обслуговування
- **Статистика** -- Перегляд статистики друку
- **Активні сповіщення** -- Інлайн-кнопки при завершенні, помилці, прогресі друку

---

## :material-cog: Налаштування

### Крок 1: Створення Telegram-бота

1. Відкрийте Telegram і знайдіть **@BotFather**
2. Надішліть `/newbot` і дотримуйтесь інструкцій
3. Оберіть ім'я та юзернейм для бота
4. Скопіюйте **токен бота**, наданий BotFather

### Крок 2: Налаштування в BamDude

1. Перейдіть до **Settings** > **Notifications**
2. Додайте провайдер сповіщень **Telegram**
3. Вставте токен бота
4. Увімкніть провайдер
5. Бот почне автоматичний полінг

### Крок 3: Авторизація чату

1. Відкрийте бота в Telegram і надішліть `/start`
2. Бот зареєструє ваш чат
3. Перейдіть до **Settings** > **Notifications** > **Telegram Chats** у BamDude
4. Авторизуйте чат і призначте групу (роль)

!!! info "Джерело токена бота"
    Токен бота зчитується з першого увімкненого провайдера сповіщень Telegram у базі даних.

!!! info "Перевірка прав на кожне повідомлення"
    Кожна команда та натискання на інлайн-кнопку проходить через `auth_middleware` перед запуском обробника. Middleware шукає чат у `telegram_chats`, визначає призначену йому групу і відхиляє дію, якщо група не має відповідного права `resource:action` (наприклад, `printers:control`, `archives:read`). Неавторизовані чати побачать "У вас немає прав" / "You don't have permission" замість виконання дії.

## :material-message-text: Команди

| Команда | Опис |
|---------|------|
| `/start` | Реєстрація чату та головне меню |
| `/printers` | Список усіх принтерів зі статусом |
| `/queue` | Перегляд черги друку (з пагінацією) |
| `/stats` | Перегляд статистики друку |
| `/help` | Показати доступні команди |

---

## :material-view-grid: Інлайн-меню

Бот використовує інлайн-кнопки для навігації замість текстових команд:

- **Список принтерів** -- Натисніть на принтер для деталей, керування або камери
- **Дії друку** -- Пауза, зупинка, відновлення з підтвердженням
- **Пресети швидкості** -- Кнопки швидкого налаштування швидкості
- **Камера** -- Запит живого знімка
- **Калібрування** -- Запуск вирівнювання столу, компенсації вібрації тощо
- **Черга** -- Пагінований перегляд черги з індикаторами статусу

## :material-image: Знімки з камери

Запитуйте знімки з камери безпосередньо в Telegram:

1. Оберіть принтер зі списку
2. Натисніть **Camera**
3. Знімок надсилається як фото-повідомлення

---

## :material-book-open: Друк з бібліотеки (Scene/FSM)

Запуск друку з файлової бібліотеки через інтерактивний потік:

1. Натисніть **Print from Library** в головному меню
2. Перегляньте файли з пагінацією
3. Оберіть файл
4. Оберіть цільовий принтер
5. Підтвердіть для друку або додавання в чергу

Потік використовує FSM (скінченний автомат) aiogram для багатокрокових взаємодій.

---

## :material-printer-3d-nozzle: Додавання принтера (Scene)

Додайте новий принтер безпосередньо з Telegram:

1. Введіть IP-адресу принтера
2. Введіть код доступу (8-значний код з мережевих налаштувань принтера)
3. BamDude підключається, автоматично визначає модель та серійний номер принтера і додає його в базу
4. Підтвердіть додавання

!!! note "Без mDNS-автодискаверингу"
    Сцена додавання принтера в Telegram явно запитує IP + код доступу -- автоматичного пошуку в LAN зсередини бота немає. Знайдіть IP на екрані принтера (Settings > Network) перед запуском потоку.

---

## :material-bell-ring: Активні сповіщення

Сповіщення, надіслані в Telegram, включають інлайн-кнопки дій:

| Сповіщення | Дії |
|-----------|-----|
| **Print Complete** | Очистити стіл |
| **Print Failed** | Очистити стіл |
| **Maintenance Due** | Позначити виконаним |
| **Print Progress** | Пауза / Зупинка |

---

## :material-translate: Інтернаціоналізація

Усі рядки інтерфейсу бота перекладаються. Рядки зберігаються в `backend/app/data/telegram_ui_{lang}.json` і доступні через `t(lang, "telegram_ui", "key")`.

Наразі підтримуються: англійська (en), українська (uk).

---

## :material-lightbulb: Поради

!!! tip "Форматування MarkdownV2"
    Весь динамічний текст, надісланий ботом, використовує формат Telegram MarkdownV2. Спеціальні символи автоматично екрануються через `escape_md()`.

!!! tip "Підтримка групових чатів"
    Бот працює як у приватних, так і в групових чатах. Кожен чат авторизується незалежно зі своїми правами.

!!! tip "Маршрутизація сповіщень"
    Різні чати можуть отримувати різні події сповіщень. Налаштуйте події сповіщень для кожного чату у веб-інтерфейсі.
