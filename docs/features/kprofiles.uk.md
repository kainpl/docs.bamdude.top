# K-профілі

K-профілі — це BamDude'ове представлення slicer-grade параметрів друку, тюнованих на принтер: pressure-advance / linear-advance значення, температури сопла/столу, flow ratio, налаштування ретракту. Профілі живуть разом з твоєю інсталяцією: бекапляться по Git на розклад, експортуються одним JSON-файлом при міграції, застосовуються per-spool з inventory-сторінки.

Ця сторінка про механіку профілів. Котушковий бік (як прив'язати профіль як override до конкретної котушки / кольору) — на сторінці [Інвентар котушок](inventory.uk.md).

## :material-cog: Де живуть профілі

| Місце | Що робить |
|---|---|
| **Profiles** у бічному меню | Список профілів. Фільтр за моделлю принтера, пошук по імені, редагування, клонування, імпорт, експорт. |
| Деталі котушки (під Inventory) | "K-profile override" — обрати профіль для використання, коли ця котушка призначена слоту AMS. Корисно для off-spec пластиків. |
| Settings → Backup → Git | Розклад бекапу профілів + cloud-profile-snapshot-ів до GitHub / GitLab. |

## :material-content-duplicate: Клонування vs редагування

Профіль може бути в трьох станах:

- **System default** — постачається з BamDude, read-only. Клонуй, щоб кастомізувати.
- **User-owned** — створений тобою, повністю редагується.
- **Imported** — підтягнутий з іншої інсталяції / Git-restore. Редагується, але import-metadata залишається — видно, звідки прийшов.

Клонування system-default дає user-owned копію з тими ж параметрами; оригінал недоторканий. Більшість BamDude-операторів закінчують з одним user-owned на пару `(модель принтера × матеріал)`.

## :material-printer-3d: Обмеження для двосопельних

H2D / H2D Pro мають друге сопло; їхні профілі повинні експортувати два паралельні набори flow / temperature параметрів. BamDude визначає двосопельність на runtime з MQTT-телеметрії — конкретно, коли принтер публікує ключ `nozzle_2` у temperature-стрімі, BamDude перемикає dual-nozzle прапор. Окремої таблиці serial-префіксів немає; детекція суто capability-based, тож будь-яка майбутня модель Bambu, що публікує `nozzle_2`, підхопиться автоматично. Після детекції BamDude:

- Фільтрує picker профілів до двосопельних, коли таргет — двосопельний принтер.
- Приховує блок UI для другого сопла на single-nozzle профілях, щоб не шуміти.
- Відмовляється присвоїти single-nozzle профіль двосопельному принтеру (і навпаки) — побачиш чітку помилку в UI, а не silent malfunction під час друку.

## :material-import: Імпорт & експорт

Три шляхи імпорту:

| Джерело | Примітки |
|---|---|
| **Single profile JSON** | Сторінка Profiles приймає upload раніше експортованого профілю. |
| **Bundle import** | Профілі + відповідні spool-overrides одним ZIP; корисно при міграції між інсталяціями. |
| **OrcaSlicer preset** | OrcaSlicer постачає filament presets, які чисто мапляться на параметри K-профілю. Кинь папку з `.json` Orca-пресетами філаменту — BamDude імпортує в один клік. |

Експорт — оберненим шляхом: single profile, full bundle або "все змінене з X" diff-bundle.

## :material-cloud-upload: Інтеграція з Git-бекапом

Settings → Backup → Git вміє пушити твій каталог профілів (плюс BamDude config snapshot) в GitHub / GitLab репо на розклад. Репо містить:

- Плоску директорію `kprofiles/` з JSON-ами профілів (один файл на профіль, назва за ID).
- `kprofiles_index.json` з кожним профілем за хешем + ім'ям + моделлю принтера, щоб partial restore був однозначним.
- Відповідні spool overrides (`spool_kprofiles/`).
- Audit-log запис на кожен scheduled-commit.

Restore з Git за замовчуванням partial: можна обрати конкретні профілі для повернення або "все, починаючи з останнього бекапу". Git-історія сама — твій version-archive — кожен commit це snapshot каталогу профілів, до якого можеш відкатитися.

## :material-account-multiple: Дозволи

| Permission | Ефект |
|---|---|
| `kprofiles:read` | Дивитися список, бачити налаштування. Дефолт для Viewers. |
| `kprofiles:create` | Створювати нові або клонувати наявні. |
| `kprofiles:update` | Редагувати user-owned / imported профілі. |
| `kprofiles:delete` | Видаляти user-owned (system-defaults ніколи не видаляються). |

Spool overrides (`spool_kprofile`) керуються `inventory:update` — тою ж поверхнею, що й редагування котушки.
