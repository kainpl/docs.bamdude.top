# Налаштування першого принтера

Цей посібник проведе вас через додавання першого принтера до BamDude.

---

## :material-list-status: Передумови

Перед додаванням принтера переконайтеся:

- [x] BamDude запущений ([Встановлення](installation.uk.md) або [Docker](docker.uk.md))
- [x] Принтер увімкнений та підключений до мережі
- [x] **SD-карта вставлена** в принтер
- [x] Developer Mode увімкнений ([інструкція](index.uk.md#developer-mode))
- [x] У вас є IP-адреса, серійний номер та код доступу

---

## :material-printer-3d-nozzle: Додавання принтера

Принтер можна додати двома шляхами: дозволити BamDude **автоматично знайти**
його в LAN (найшвидше) або вписати IP / access-code / serial **вручну**.

### Варіант A — Автодискаверинг у LAN

1. Відкрийте BamDude → **Printers** → **:material-plus: Add Printer** →
   **Discover on LAN**.
2. Виберіть метод дискаверингу під ваш інстал:

    === "Native install"

        BamDude слухає **SSDP** broadcast-и. Принтери Bambu Lab анонсують
        себе в LAN кожні кілька секунд і автоматично з'являються у списку
        дискаверингу. Натисніть **Start**, почекайте ~10 секунд, виберіть
        свій принтер зі списку.

    === "Docker (host network)"

        SSDP працює, поки контейнер на `network_mode: host` (див.
        [Docker → Host-mode compose](docker.uk.md)). Той самий потік, що й
        нативний інстал.

    === "Docker (bridge network)"

        SSDP-multicast не перетинає bridge, тож BamDude перемикається на
        **subnet scan**. Введіть CIDR вашої LAN (наприклад,
        `192.168.1.0/24`) і натисніть **Scan** — BamDude перевіряє кожен
        хост на портах MQTT/FTPS Bambu і виводить кожен знайдений принтер.
        На `/24` сканування завершується менш ніж за хвилину.

3. Натисніть на принтер у списку результатів. BamDude підставить **IP** та
   **Serial** автоматично — лишиться лише ввести **Access Code**.
4. Перейдіть до *Кроку 3 — Збережіть та підключіть*.

!!! info "Право для дискаверингу"
    Маршрути дискаверингу потребують права `discovery:scan`. Дефолтні групи
    **Administrators** та **Operators** його мають; **Viewers** — ні.

### Варіант B — Ручне введення

Використовуйте, коли дискаверинг недоступний (інший VLAN, mDNS заблоковано,
обмежене середовище) або ви вже знаєте дані підключення принтера.

1. Відкрийте BamDude у браузері
2. Перейдіть на сторінку **Printers**
3. Натисніть кнопку **:material-plus: Add Printer**
4. Виберіть **Enter manually**

### Крок 2: Введіть дані принтера

| Поле | Опис | Приклад |
|------|------|---------|
| **Name** | Зрозуміла назва принтера | `Workshop X1C` |
| **Model** | Виберіть модель зі списку (X1C, P1S, A1 Mini, H2D, …). BamDude використовує її, щоб увімкнути model-specific фічі (двері-сенсор, dual nozzle, AMS-HT) ще до того, як MQTT це підтвердить. | `X1 Carbon` |
| **IP Address** | Локальна IP-адреса принтера (автодискаверинг заповнює сам) | `192.168.1.100` |
| **Access Code** | 8-символьний код з Developer Mode | `12345678` |
| **Serial Number** | Серійний номер принтера (автодискаверинг заповнює сам) | `01P00A000000001` |

!!! tip "Вибір моделі важливий"
    BamDude виводить кілька можливостей з моделі — детекцію стану дверей,
    UI dual nozzle, рендер AMS-HT слотів, фільтрацію завдань обслуговування
    (carbon/steel rod проти linear rail). Помилковий вибір моделі сховає
    або покаже не ті фічі. Якщо не впевнені, до якого класу належить ваш
    принтер, перевірте
    [матрицю можливостей](../reference/printers.uk.md#bamdude-capability-matrix).

### Крок 3: Збережіть та підключіть

1. Натисніть **Save**
2. BamDude спробує підключитися до принтера
3. Дочекайтеся, поки індикатор з'єднання стане зеленим

## :material-cloud-outline: Bambu Cloud (опціонально, per-user)

Прив'язка вашого Bambu-аккаунта вмикає cloud-only фічі: перевірку оновлень
прошивки, cloud-каталог філаментів, slicer-setting профілі. Кожен
**користувач** BamDude зберігає власні cloud-credentials — адміни не діляться
своїм Bambu-аккаунтом з операторами чи viewers.

1. Відкрийте **Settings → Cloud** під логіном того користувача, якого
   хочете прив'язати.
2. Виберіть **регіон** — `global` (дефолт, EU/US/AS) або `china`. Регіон
   мапиться на інший endpoint Bambu Cloud; помилковий вибір видасть
   `Login failed`.
3. Введіть email + password. Якщо у вашому Bambu-аккаунті увімкнено 2FA,
   далі система запитає код підтвердження.
4. Токен зберігається зашифрованим на рядку користувача. Інші користувачі
   на тому ж BamDude-інсталі його **не бачать** і **не використовують**.

!!! info "Auth вимкнено?"
    На інсталях, де auth вимкнено (рідкість — auth увімкнено за замовчуванням
    з 0.4.0), cloud-токен фолбекає на глобальну таблицю `Settings` замість
    per-user сховища.

## :material-alert-circle: Проблеми з підключенням?

1. **Developer Mode увімкнений?** Перевірте LAN Only Mode та Developer Mode
2. **Правильна IP-адреса?** Перевірте в мережевих налаштуваннях принтера
3. **Актуальний код доступу?** Коди змінюються при перемиканні Developer Mode
4. **Та сама мережа?** Сервер BamDude та принтер мають бути в одній локальній мережі
5. **Правила файрволу?** Переконайтеся, що порти 8883 (MQTT) та 990 (FTPS) відкриті

Дивіться повний [Посібник з вирішення проблем](../reference/troubleshooting.md).
